# Agents

This project uses the Payload CMS skill at `.agents/skills/payload/`.
Start with `.agents/skills/payload/SKILL.md` for a quick reference, then see `.agents/skills/payload/reference/` for detailed docs.

## Project

Payload CMS 3.84.1 + Next.js 16.2.6 + PostgreSQL (Neon). ESM (`"type": "module"`).
Package manager: **pnpm** (^9 || ^10). `.npmrc` sets `legacy-peer-deps=true`.

## Commands

| Command | Description |
|---|---|
| `pnpm dev` | Start dev server |
| `pnpm devsafe` | Clear `.next` then start dev server |
| `pnpm build` | Production build (8GB memory limit) |
| `pnpm start` | Start production server |
| `pnpm lint` | ESLint |
| `pnpm test:int` | Run integration tests (vitest) |
| `pnpm test:e2e` | Run e2e tests (Playwright) |
| `pnpm test` | Run both int + e2e tests sequentially |
| `pnpm generate:types` | Regenerate Payload TypeScript types |
| `pnpm generate:importmap` | Regenerate Payload import map |

## Architecture

- **Payload config**: `src/payload.config.ts` — aliased as `@payload-config`
- **Collections**: `src/collections/` — Users, Media, WasteCategories, Reports
- **Globals**: `src/globals/` — SiteSettings
- **App routes**: `src/app/(frontend)/`, `src/app/(payload)/`, `src/app/my-route/`
- **Path aliases**: `@/*` → `./src/*`, `@payload-config` → `./src/payload.config.ts`

## Generated files (do not edit)

- `src/payload-types.ts` — generated by `pnpm generate:types`
- `src/payload-generated-schema.ts` — generated by Payload
- Both are ignored by ESLint

## Testing

- **Integration**: vitest with jsdom, pattern `tests/int/**/*.int.spec.ts`
- **E2E**: Playwright (Chromium only), tests in `tests/e2e/`
- E2E tests auto-start dev server via `pnpm dev` on `http://localhost:3000`
- Test helpers in `tests/helpers/`

## Style

- Prettier: single quotes, no semicolons, trailing commas, 100 char print width
- ESLint extends `next/core-web-vitals` + `next/typescript` with relaxed rules (unused vars prefixed with `_` are ignored)

## Database

Uses PostgreSQL via `@payloadcms/db-postgres` with a Neon connection string.
Despite `.env.example` showing MongoDB, the actual config uses Postgres.

## MCP

Stitch MCP server configured in `opencode.json` (remote, requires `STITCH_API_KEY` env var).
